Примеры возвращаемого ответа:
{
    "status": 200,
    "message": "success1",
    "result": null,
    "timestamp": 1705043555672
}

{
    "status": 200,
    "message": "success",
    "result": {
        "status": 3,
        "warrantyDuration": "2020-09-23",
        "brand": "TECNO",
        "model": "CF7",
        "imei": [
            "359050097375085",
            "359050097375093",
            "",
            ""
        ],
        "activeTime": "2019-08-23 00:00:00"
    },
    "timestamp": 1705043556752
}

message -- при успешном получении ответа должен быть "success". Стоит добавить обработку, чтобы при иных результатах запрос повторялся, причем ограниченное количество раз
result -- содержит результат запроса. Данные из него записываются в таблицу.
	status -- не до конца понятно, за что отвечает этот параметр. Значение 2 возвращается при отсутствии активации, 3 - если устройство активировано, независимо от того на гарантии оно или нет. Неизвестно, есть ли другие значения и за что они могут отвечать
	warrantyDuration -- дата истечения гарантии
	brand, model и imei -- говорят сами за себя
	activeTime -- дата активации утройства

Скрипт обработки запроса:
1. Проверка параметра message
	- Если параметр message равен "success", начинаем обработку результата
	- Если параметр иной, повторяем запрос (для этого потребуется значения imei и количество запросов, иначе цикл может стать бесконечным)
2. Обработка result
	- Если status равен 2, то достаточно записать в таблицу imei и указать статус "не активирован"
	- Если status равен 3, требуется сравнить дату истечения гарантии с сегодняшней датой
		- Если дата истечения гарантии меньше сегодняшней, то возвращаем статус "истекла"
		- В иных случаях возвращаем "активна"
		- В обоих случаях помимо imei и статуса также возвращаем дату активации и дату истечения гарантии
